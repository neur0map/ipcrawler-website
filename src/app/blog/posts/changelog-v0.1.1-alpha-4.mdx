## IPCrawler v0.1.1-alpha-4

ðŸ”§ **SmartList 2.0: Improved Wordlist Recommendations**

This release introduces SmartList 2.0, a redesigned recommendation engine that aims to provide more accurate and context-aware wordlist selection during security assessments.

## ðŸš€ What's New in SmartList 2.0

### Expanded Technology Detection
SmartList 2.0 now recognizes 34+ technologies across 9 categories, expanding from the previous 10 technologies. The system can better identify modern applications and services in target environments.

**New Technology Categories:**
- **CI/CD Platforms**: GitLab, Jenkins, GitHub Enterprise, Bitbucket, Jira, Confluence
- **Container Orchestration**: Docker, Kubernetes, OpenShift  
- **Modern Frameworks**: Laravel, Symfony, React, Angular, Vue.js
- **Monitoring Tools**: Splunk, Elastic Stack, Logstash, Datadog, Grafana, Prometheus
- **Database Systems**: MySQL, PostgreSQL, MongoDB, Redis, Elasticsearch

### Improved Recommendation Logic
The new scoring system considers multiple factors when suggesting wordlists:

- **Technology Matching**: Prioritizes wordlists that match detected technologies
- **Confidence Levels**: Adjusts recommendations based on detection confidence  
- **Scan Context**: Considers the type of assessment being performed
- **Quality Metrics**: Uses historical effectiveness data to guide selection

### Better Diversity Controls
To reduce repetitive recommendations, the system now includes:

- **Dynamic Selection**: Varies recommendations based on context and timing
- **Balanced Approach**: Mixes targeted and exploratory wordlists appropriately
- **Reduced Redundancy**: Limits similar wordlists in single scans
- **Optimized Coverage**: Balances comprehensive and focused approaches

## ðŸ”§ Technical Changes

SmartList 2.0 uses a database-driven architecture that removes hardcoded mappings. Key improvements include:

- **Multi-Factor Scoring**: Considers multiple contextual factors for each recommendation
- **Quality-Based Filtering**: Uses accuracy and specificity metrics in selection
- **Extensible Design**: Easier to add new technologies and wordlists
- **Backward Compatibility**: Existing workflows continue to function
